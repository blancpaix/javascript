매우 빠르고 유연한 인메모리 데이터 구조의 저장소 Redis
종종 데이터베이스 또는 캐시 서버로 사용됨
많은 기능 중 중앙 집중식 Pub/Sub 메시지 교환 패턴을 구현하도록 특별히 설계된 명령어 한쌍이 있지 ㅎㅎㅎ

Redis 메시지 브로커 기능, 매우 간단하고 기본적!!
캐시 서버나 세션 데이터 저장소 등 기존 인프라에서도 사용 가능
속도와 유연성 덕분에 분산 시스템에서 데이터 공유하는데 매우 널리 사용
간단하고 즉각적인 선택으로 전용 메시지 브로커 설치 및 유지보수가 필요없어짐, Redis 사용하세요!

예제를 통해 알아봅시다

https://redis.io/topics/quickstart

일단 이거 먼저 설치를 해주시고
https://github.com/tporadowski/redis
윈도우를 지원안해서 따로 이렇게 올리는듯 오픈소스 쩌네
레디스 서버를 그냥 실행시키면 됨

간단하게
클라이언트 - 서버1 => 레디스 =>>> [서버 1,2,3]  =>> [클라 2,3]
메시지 서버 전송 => 메시지 브로커 게시 => 브로커가 모든 구독자에(인스턴스)게 메시지 발송 => 각 인스턴스에서 연결된 클라이언트에게 전송


!! Redis 사용하면서 "chat.nodejs" 와 같은 문자열로 식별되는 채널을 게시하고 구독할 수있음
  globstyle("*") 패턴을 사용해서 여러 채널 (Ex) chat.*)과 일치하는 구독을 정의할 수도 잇음
